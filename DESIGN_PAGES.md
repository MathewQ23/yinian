# 一念 6 个页面设计文档

基于 `/home/mathew/code/yinian/design/` 下 6 张页面设计图整理。

这组设计共同定义的是新版「一念」桌面端产品：从轻量想法记录，升级为围绕「工作线 / 记录 / 来源 / 关系 / 待探索」组织的个人知识工作台。

整体设计关键词：

- 温暖、克制、可信
- 桌面端效率工具
- 米白背景 + 深棕主色
- 卡片化但不过度装饰
- 记录优先，结构从记录中自然生长
- 强调当前状态、下一步、来源、引用与演化关系

---

## 全局设计系统

### 1. 全局导航

所有页面共享同一套顶部导航：

```text
一念 Logo   今天   工作线   待探索   全部记录       搜索框       + 快速记录
```

#### 导航规则

- Logo 位于左侧，深棕色圆角矩形，白色「一念」文字。
- 一级导航包含 4 个页面：
  - 今天
  - 工作线
  - 待探索
  - 全部记录
- 当前页面使用：
  - 更深文字
  - 较高字重
  - 底部深色短下划线
- 右侧搜索框为全局搜索：
  - 搜索记录
  - 搜索工作线
  - 搜索来源
- `+ 快速记录` 是全局最高频操作，始终位于右上角。

### 5. 核心信息模型

这 6 个页面共同确认的一念核心实体：

```text
Thread 工作线
Entry 记录
Source 来源
Relation 关系
ExploreItem 待探索内容
```

#### 工作线 Thread

表示一件持续推进的事情。

包含：

- 名称
- 描述
- 当前状态
- 当前阶段
- 当前阻塞
- 下一步
- 最近更新
- 记录数
- 问题数
- 来源数

#### 记录 Entry

表示某个时间点发生的思考、进展、问题、方案、决策或线索。

类型：

- 进展
- 问题
- 方案
- 决策
- 线索
- 普通记录

#### 来源 Source

表示触发记录或探索的外部材料。

类型：

- 链接
- 图片
- 视频
- GitHub
- Figma
- 本地文件
- 文档 / 文章 / 论文

#### 关系 Relation

表示记录之间的连接。

核心关系：

- 来源
- 引用
- 延伸
- 关联
- 解决

#### 待探索 ExploreItem

表示暂时值得关注、后续研究的内容。

状态：

- 待探索
- 探索中
- 已验证
- 已采用
- 放弃

# 页面 1：首页 / 今天

对应设计图：`ChatGPT Image 2026年7月8日 22_11_17 (1).png`

## 页面定位

「今天」是打开频率最高的首页工作台。

核心问题：

> 今天我应该继续什么？刚刚发生了什么？哪些事情值得记录？

它不是全部信息的入口，而是当日工作推进的总览与快速记录中心。

## 页面结构

```text
顶部导航
页面问候与今日摘要
快速记录 | 今日概览
继续推进工作线
今日记录 | 待探索
```

## 主要模块

### 1. 问候区

内容：

```text
晚上好
今天推进了 4 条工作线 · 记录 8 条 · 解决 2 个问题
```

设计规则：

- 标题大而稳，约 32px。
- 副标题用于反馈当天产出。
- 不做大 Hero，不占用过多工作区。

### 2. 快速记录

首页最重要模块。

结构：

```text
快速记录
[刚刚发生了什么？写下一条进展、问题、方案、线索...]
@ 工作线   + 来源   / 类型
记录时间 2026 / 07 / 08 21:18              保存 Ctrl+Enter
```

设计原则：

- 输入框是视觉中心。
- 默认只需要输入正文。
- 工作线、来源、类型都是可选增强。
- `Ctrl + Enter` 保存必须保留。
- 保存按钮使用深棕主按钮。

### 3. 今日概览

统计项：

- 工作线 4
- 问题 2
- 待探索 5
- 已解决 2

下方显示最近 7 天记录趋势柱状图。

设计规则：

- 统计卡片 2x2。
- 当前日期柱子高亮。
- 统计项未来可点击筛选。

### 4. 继续推进

展示 3 条最重要工作线：

- 一念产品
- RoArm-M3
- PDF 翻译

每张卡片包含：

```text
图标 + 名称 + 状态
最近更新
当前阶段
阻塞点 / 下一步
查看工作线 →
```

设计规则：

- 重点显示“当前阶段”和“下一步”。
- 不显示百分比进度。
- 阻塞项使用橙色状态。

### 5. 今日记录

时间线列表，展示当天记录。

记录类型包括：

- 进展
- 问题
- 线索
- 方案
- 决策

设计规则：

- 左侧时间 + 节点。
- 类型标签低饱和。
- 底部提供 `查看全部记录 →`。

### 6. 待探索

展示 4 个待探索内容：

- browser-harness
- GSAP Skills
- PDF Layout Parser
- Eye-to-Hand Calibration

设计规则：

- 2x2 小卡片。
- 展示标题、来源、状态。
- 底部提供 `查看全部待探索 →`。

## 交互

- 点击顶部导航切换页面。
- 点击 `+ 快速记录` 或首页快速记录保存新记录。
- 点击工作线卡片进入工作线详情。
- 点击今日记录进入记录详情。
- 点击待探索卡片进入待探索详情。

## 响应式

移动端顺序：

```text
问候区
快速记录
今日概览
继续推进
今日记录
待探索
```

# 页面 2：工作线列表

对应设计图：`ChatGPT Image 2026年7月8日 22_11_17 (2).png`

## 页面定位

「工作线」用于查看所有正在推进、暂停和已完成的长期事项。

核心问题：

> 我现在有多少事情在推进？每条线做到哪里？卡在哪里？下一步是什么？

## 页面结构

```text
顶部导航
页面标题 + 描述 + 状态筛选 + 新建工作线
工作线卡片网格 | 本周概况侧栏
```

## 主要模块

### 1. 页面标题区

```text
工作线
查看所有正在推进、暂停和已完成的工作线，清楚知道每件事现在做到哪里。
```

右侧：

- `进行中 6`
- `暂停 2`
- `已完成 12`
- `+ 新建工作线`

设计规则：

- 状态筛选使用 segmented control。
- 新建工作线是页面内操作，权重低于全局快速记录。

### 2. 工作线卡片网格

两列网格，展示 6 条工作线：

- 一念产品
- RoArm-M3
- PDF 翻译
- browser-harness 调研
- 时间记录小工具
- 视觉标定研究

单张卡片结构：

```text
图标  工作线名称  状态标签
工作线描述

当前阶段         当前阻塞         下一步
阶段标签组       阻塞说明         下一步说明

最近更新  记录数  问题数  来源数       查看详情 →
```

### 3. 阶段标签

阶段标签分三类：

- 已完成：绿色，对勾
- 当前阶段：橙色，实心点
- 未开始：灰色，空心点

设计规则：

- 阶段不是进度百分比。
- 阶段是用户真实推进过程中的离散状态。

### 4. 本周概况侧栏

统计项：

- 活跃工作线 6
- 本周更新 28
- 未解决问题 3
- 待推进下一步 5

设计规则：

- 右侧固定宽度。
- 数字大，说明小。
- 可点击进入对应筛选。

## 交互

- 状态 Tab 筛选工作线。
- 点击卡片或 `查看详情 →` 进入工作线详情。
- 点击 `+ 新建工作线` 打开创建流程。
- 点击侧栏统计项筛选对应集合。

## 实现要点

工作线卡片必须优先展示：

1. 当前阶段
2. 当前阻塞
3. 下一步
4. 最近更新

不要在卡片上显示“进度 72%”这类虚假精确度。

# 页面 3：工作线详情 / 进展

对应设计图：`ChatGPT Image 2026年7月8日 22_11_17 (3).png`

## 页面定位

某条工作线的详情页，当前停留在「进展」Tab。

核心问题：

> 这条工作线最近发生了什么？当前状态怎样？下一步是什么？有哪些来源和关键决定？

## 页面结构

```text
顶部导航
面包屑 + 工作线标题 + 状态
三张摘要卡：当前阶段 / 下一步 / 最近更新
页内 Tab：概览 / 进展 / 链条 / 资料
左侧进展流 | 右侧状态侧栏
```

## 主要模块

### 1. 工作线标题区

```text
工作线 / 一念产品
一念产品   进行中
记录工作和思考如何持续推进，并让结构从记录中自然生长出来。
```

设计规则：

- 面包屑弱化。
- 标题最大。
- 状态标签紧跟标题。
- 描述说明工作线目标。

### 2. 摘要卡

三张横向卡片：

- 当前阶段：重新定义产品模型
- 下一步：完成首页信息架构
- 最近更新：今天 10:33

设计规则：

- 作为 5 秒概览。
- 图标 + 文案。
- 不承担复杂操作。

### 3. 页内 Tab

```text
概览    进展    链条    资料
```

当前：进展。

设计规则：

- 只用于同一工作线内部切换。
- 当前 Tab 使用下划线。

### 4. 进展流

按日期分组的时间线。

记录类型：

- 进展
- 问题
- 线索
- 方案
- 决策
- 普通记录

单条记录结构：

```text
时间  类型标签  内容  可选来源
```

示例：

```text
10:33  进展  重新梳理产品定位，从“想法链”调整到“工作与思考进展记录”
09:52  问题  发现单一链条无法表达多条工作同时推进
09:20  线索  研究几款视觉知识工具，记录值得借鉴的点  来源：Heptabase 文章
```

设计规则：

- 时间线强调“过程”。
- 记录卡不应过重。
- 来源以小胶囊展示。

### 5. 右侧状态侧栏

模块：

1. 当前状态
2. 下一步
3. 相关来源
4. 关键决定
5. 操作按钮

#### 当前状态

包含：

- 进行中
- 当前阶段：工作线设计
- 阻塞项：1
- 待探索：3
- 已解决问题：4

#### 下一步

待办清单：

- 完成首页信息架构
- 梳理核心概念与关系
- 输出工作线使用指南 v0.1

#### 相关来源

如：

- Heptabase 文章
- Memos 笔记

#### 关键决定

沉淀高价值结论，例如：

- 保留原有 UI 资产，重组信息架构以降低迁移成本。
- 增加 Thread 作为一级实体，支持多条线索并行推进。

#### 操作按钮

- `+ 继续记录`：主按钮
- `查看链条`：次按钮

## 交互

- 点击 Tab 切换详情视角。
- 点击进展记录打开记录详情。
- 点击来源进入来源详情。
- 点击下一步 checkbox 可完成任务。
- 点击继续记录时，新记录默认归属当前工作线。

# 页面 4：待探索

对应设计图：`ChatGPT Image 2026年7月8日 22_11_17 (4).png`

## 页面定位

「待探索」用于管理暂时值得关注但还没深入处理的内容。

核心问题：

> 哪些东西值得以后研究？它们现在处于什么探索状态？是否应该加入某条工作线？

它不是收藏夹，而是研究对象池。

## 页面结构

```text
顶部导航
页面标题 + 状态 Tab
左侧筛选 | 中间待探索内容 | 右侧探索详情
```

## 主要模块

### 1. 页面标题区

```text
待探索
先记下来，等有时间再继续研究。把值得进一步看的内容和后续结论放在一起。
```

### 2. 状态 Tab

```text
待探索 12    探索中 4    已验证 8    已采用 6    放弃 3
```

设计规则：

- 状态 Tab 表示探索生命周期。
- 当前状态高亮。
- 这是一级流程筛选，不是普通标签筛选。

### 3. 左侧筛选

分组：

#### 来源类型

- GitHub
- 网页
- 视频
- 图片

#### 标签

- 机械臂
- 设计
- 产品
- 灵感
- 技术

#### 工作线归属

- 未归属
- 一念产品
- RoArm-M3
- PDF 翻译

设计规则：

- 筛选分组可折叠。
- 每项显示数量。
- 当前筛选要有明确选中态。

### 4. 中间内容列表

标题：待探索内容。

右上角：网格 / 列表视图切换。

卡片内容：

- 图标 / 缩略图
- 标题
- 来源类型与域名
- 描述
- 保存时间
- 状态标签
- 查看详情

示例内容：

- browser-harness
- GSAP Skills
- PDF Layout Parser
- Eye-to-Hand Calibration
- Open source whiteboard library
- Realtime data sync article

选中卡片使用橙色描边。

### 5. 右侧探索详情

当前选中：browser-harness。

详情结构：

1. 基础信息
2. 保存时间
3. 保存原因
4. 标签
5. 预览
6. 探索记录
7. 关联工作线
8. 操作按钮

#### 预览

针对 GitHub / 网页 / 文档展示摘要：

- 摘要文字
- star 数
- 技术栈
- License

#### 探索记录

用时间线记录探索过程：

- 跑通示例项目
- API 与现有系统不兼容
- 结论：暂不采用当前方案

#### 操作按钮

- `开始探索 / 继续探索`：主按钮
- `加入工作线`：次按钮

## 交互

- 点击状态 Tab 改变探索阶段筛选。
- 点击左侧筛选缩小结果。
- 点击卡片在右侧展示详情。
- 点击开始探索，将状态从待探索变为探索中。
- 点击加入工作线，建立 ExploreItem 与 Thread 的关联。

## 设计原则

待探索内容必须记录“为什么值得看”，否则会退化成普通收藏夹。

- 本月

# 设计验收标准

实现时需要满足：

1. 6 个页面共享同一导航和设计语言。
2. 米白背景、深棕主色、低饱和状态色保持一致。
3. 卡片圆角控制在 10-14px，不要过度圆角。
4. 阴影必须克制，不做大面积漂浮卡片。
5. 记录输入路径不能比发一条微信消息复杂。
6. 工作线卡片必须展示当前阶段、阻塞、下一步，而不是进度百分比。
7. 待探索不是收藏夹，必须有保存原因、探索记录和结论状态。
8. 全部记录必须能通过时间、工作线、类型、来源、状态筛选。
9. 链条页必须清楚区分延伸关系和关联关系。
10. 所有页面必须有移动端降级方案，但桌面端是主体验。
